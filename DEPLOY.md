# Deployment Guide: Vercel + Render

This app uses:
- **Vercel** – Frontend (React/Vite)
- **Render** – Backend API + Telegram notifications

---

## Step 1: Deploy Backend to Render

1. Push this repo to GitHub (see Step 3).
2. Go to [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
3. Connect your GitHub repo.
4. Render will read `render.yaml`. Confirm the service and click **Apply**.
5. In the service **Environment** tab, set:
   - `ADMIN_PASSWORD` – Your admin login
   - `TELEGRAM_BOT_TOKEN` – From [@BotFather](https://t.me/BotFather)
   - `TELEGRAM_CHAT_IDS` – Your chat ID(s), comma-separated
   - `SUPABASE_URL` – Your Supabase project URL
   - `SUPABASE_SERVICE_ROLE_KEY` – Supabase service role key
   - `STRIPE_SECRET_KEY` – Stripe secret key
   - `APP_URL` – Your Vercel URL (e.g. `https://your-app.vercel.app`)
6. Deploy. Copy the Render URL (e.g. `https://speedy-tags-api.onrender.com`).

---

## Step 2: Deploy Frontend to Vercel

1. Install Vercel CLI: `npm i -g vercel`
2. From project root:
   ```bash
   vercel --prod
   ```
3. When prompted, link to your Vercel project or create one.
4. In [Vercel Dashboard](https://vercel.com/dashboard) → Project → **Settings** → **Environment Variables**, add:
   - `VITE_API_URL` = Your Render URL (e.g. `https://speedy-tags-api.onrender.com`)
5. Redeploy so the build picks up `VITE_API_URL`:
   ```bash
   vercel --prod
   ```
   Or trigger a new deployment from the Vercel dashboard.

## Environment Summary

| Var | Where | Description |
|-----|-------|-------------|
| `VITE_API_URL` | Vercel | Render backend URL (no trailing slash) |
| `APP_URL` | Render | Vercel frontend URL (for Stripe redirects) |
| `ADMIN_PASSWORD` | Render | Admin dashboard password |
| `JWT_SECRET` | Render | Auto-generated by Render |
| `TELEGRAM_BOT_TOKEN` | Render | For order notifications |
| `TELEGRAM_CHAT_IDS` | Render | Comma-separated chat IDs |
| `SUPABASE_URL` | Render | Supabase project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Render | Supabase service key |
| `STRIPE_SECRET_KEY` | Render | Stripe live secret key |

---

## Supabase Setup

1. Run `supabase/setup.sql` in Supabase SQL Editor.
2. In Storage, create a public bucket named **order-documents**.

---

## Single-Region Option (Render Only)

To run everything on Render (no Vercel):
- Deploy only the Render service.
- Set `APP_URL` to your Render URL.
- Render serves both API and frontend; no `VITE_API_URL` needed.
